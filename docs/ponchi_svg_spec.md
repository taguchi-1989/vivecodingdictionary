# 擬人化ポンチ絵 SVG 仕様 v1

*2026-04-25 作成。量産前の draft 用 SVG の見た目を揃える仕様。本番イラストは後日画像生成で差し替え、本仕様は「イメージが湧く叩き」を作るためのもの。*

**原則**: LLM が生成しても人が描いても、`drafts/ponchi/{entry_id}.svg` に保存するものは**黒い手書き風の線画**で統一する。

---

## 1. 物理仕様（固定値）

| 項目 | 値 |
|---|---|
| `viewBox` | `0 0 192 192`（1:1 正方形、`.ponchi-icon` の 200px 円に内接する 192×192） |
| 幅・高さ | SVG ルートに width/height **指定しない**（CSS で親サイズに追従） |
| 背景 | `#F3F6FB`（`--paper-2`）の薄塗り `<rect>` を 1 枚敷く。任意（なしでも可） |
| 色 | **黒 `#1A1A1A` のみ**。塗りつぶし禁止、線画のみ |
| `stroke` | `#1A1A1A` 固定 |
| `stroke-width` | **1.5〜2.4** の範囲で**敢えてばらつかせる**（手書きっぽさ） |
| `stroke-linecap` | `round` 固定 |
| `stroke-linejoin` | `round` 固定 |
| `fill` | 要素に対しては `none`（塗らない）。例外: 強調の黒丸（目・点）のみ `fill="#1A1A1A"` |

## 2. 手書きらしさの作り方

**LLM が生成すると幾何学的に綺麗になりすぎる**ため、以下を強く意識させる:

1. **直線を避ける**. 真っ直ぐな `<line>` ではなく、軽く波打った `<path>` を使う
   - 悪い例: `<line x1="0" y1="100" x2="200" y2="100"/>`
   - 良い例: `<path d="M 0 100 Q 50 98 100 101 T 200 99"/>`
2. **円を避けて楕円の近似を使う**. `<circle>` ではなく `<path>` で閉曲線を描き、完璧な円にしない
3. **線を重ねて「たどたどしさ」を出す**. 同じ輪郭を 2〜3 回軽くずらして描いても良い（鉛筆のアタリ線）
4. **角度を揃えない**. 45 度きっちりの腕／体躯ではなく、3〜5 度ずらす

参考実装（手書きフィルタ）は SVG 組み込みの `<filter>` を使える:

```xml
<defs>
  <filter id="sketch" x="-5%" y="-5%" width="110%" height="110%">
    <feTurbulence type="fractalNoise" baseFrequency="0.02" numOctaves="2" seed="3"/>
    <feDisplacementMap in="SourceGraphic" scale="1.8"/>
  </filter>
</defs>
<g filter="url(#sketch)" stroke="#1A1A1A" stroke-width="2" fill="none" stroke-linecap="round">
  <!-- ここに線画を描く -->
</g>
```

`feDisplacementMap` の `scale` を 1.5〜2.5 で調整すると、綺麗な線が"揺らいだ"見た目になる。これが最も手軽で効果的な手書き化。

## 3. 描くべき要素（誌面ポンチ絵メモを読む）

各エントリの markdown 末尾にある `## 誌面ポンチ絵メモ` の「描く内容」「登場人物」「吹き出し」を必ず参照する。

### 3-1. 人物の描き方

- **頭**: 円に近い閉曲線（完璧な円にしない）
- **体**: 肩幅を取った長方形 or 台形。服の襟や袖口を 1 本の短い線で示唆
- **髪型 / 装備で属性を示す**:
  - 一般ユーザー: 短めの髪、素の服
  - 業務担当: ネクタイ or 書類を抱える
  - 開発者: ラップトップ or ターミナル画面
- **顔**: 目は小さな **●** 2 つ、口は 1 本の曲線。鼻は描かない or 短線 1 本
- **手足**: 線のみ。指は描かない（手は球 or 四角で表現）

### 3-2. 吹き出しの描き方

- 丸みのある四角 + 尻尾 1 本
- 中の文字は**最小限（10 字以内）**。caption に書かれたセリフから抜粋
- 文字は `font-family` で日本語指定、ただし可読性のため **14-18px 程度**

### 3-3. 小物・道具

- スマホ: 角丸長方形 + 内側に短線
- PC: 台形（閉じたノート PC の俯瞰）
- 書類: A4 縦 + 短い横線 2 本
- 雲 / アイデア: 雲の輪郭のみ、中は空

---

## 4. 禁則

- ❌ **カラー使用**. 黒一色以外禁止
- ❌ **塗りつぶし**. 線画のみ（例外: 目の黒丸のみ）
- ❌ **影・グラデーション**. 単純な線のみ
- ❌ **幾何学的完璧さ**. 真円・水平線・垂直線を避ける
- ❌ **ロゴ・商標**. 「ChatGPT」ロゴや「OpenAI」マークを直接描かない。一般的なアイコンで示唆
- ❌ **実在人物の肖像**. 特定の個人を再現しない（C 章の個人エントリでも抽象化）

## 5. 良い例と悪い例

### 良い例（手書き風、人物の 1 コマ）

```xml
<svg viewBox="0 0 192 192" xmlns="http://www.w3.org/2000/svg">
  <defs>
    <filter id="sketch">
      <feTurbulence type="fractalNoise" baseFrequency="0.02" numOctaves="2" seed="3"/>
      <feDisplacementMap in="SourceGraphic" scale="1.8"/>
    </filter>
  </defs>
  <rect width="192" height="192" fill="#F3F6FB"/>
  <g filter="url(#sketch)" stroke="#1A1A1A" stroke-width="2" stroke-linecap="round" stroke-linejoin="round" fill="none">
    <!-- 頭 -->
    <path d="M 96 60 C 114 60 120 76 118 88 C 116 100 106 108 96 108 C 86 108 76 100 74 88 C 72 76 78 60 96 60 Z"/>
    <!-- 体 -->
    <path d="M 72 110 L 70 150 L 122 148 L 120 108"/>
    <!-- 腕（少し傾ける） -->
    <path d="M 72 118 Q 60 130 56 148"/>
    <path d="M 120 116 Q 138 126 142 146"/>
    <!-- 足 -->
    <path d="M 80 150 L 82 180"/>
    <path d="M 108 150 L 110 180"/>
    <!-- 目 -->
    <circle cx="88" cy="84" r="1.8" fill="#1A1A1A"/>
    <circle cx="104" cy="84" r="1.8" fill="#1A1A1A"/>
    <!-- 口 -->
    <path d="M 90 94 Q 96 98 102 94"/>
  </g>
</svg>
```

### 悪い例（避けたい "ロゴ" 的整った線画）

- `<circle>` + `<rect>` だけの幾何学的な構成
- stroke-width が完全に一定
- 一切の filter なし
- 黒以外の色（青や緑）
- 「OpenAI」「ChatGPT」ロゴの流用

## 6. 出力先と命名

- **ディレクトリ**: `drafts/ponchi/`
- **ファイル名**: `{entry_id}.svg`（例: `B-3.svg`, `D-11.svg`, `A-1.svg`）
- 1 エントリ 1 SVG 固定（複数シーンがあってもこの draft では 1 枚に集約）

preview_gen.py は `drafts/ponchi/{id}.svg` が存在すれば `.ponchi-icon` の中身を placeholder SVG からこちらに差し替えて描画する。

## 7. 本番イラストへの引き継ぎ

本 SVG は**画像生成 or 手描き差し替えの下書き**として機能する。本番イラストレーター or 画像生成モデルには以下を渡す:

- 本 SVG（draft）
- `## 誌面ポンチ絵メモ` の本文
- 本スペック（`docs/ponchi_svg_spec.md`）

本番は本 SVG をそのまま使わず、絵柄・線の質感・キャラクター表情などを上質化して差し替える想定。描く内容・構図は本 SVG を参照する。
