## 基本トーン

- 難しすぎず、しかし幼稚にしない
- 読者を馬鹿にしない
- 結論から書く
- 専門用語を消すのではなく、足場をつくる
- 事実と主観を混ぜない

## 文体

- **「です・ます」調で書く**（読者と近い距離を保つ）
- 一文を長くしすぎない
- 「つまり」「たとえば」を使って橋をかける
- カタカナ語は初出で日本語訳・役割を補う（例：Context ＝ 文脈）
- 断定できないものは「ことがあります」「場合があります」と書く
- 比喩は便利だが、仕様説明の代わりにしない

## 避ける書き方

- 「初心者でもわかる」のような上から目線
- 最新、最強、完全、絶対などの強い断定
- ベンチマークだけで優劣を決める説明
- 公式仕様と自分の体感を同じ段落で混ぜる説明
- 長いコードで誌面を埋める説明

## 用語の扱い

- サービス名、モデル名、会社名、言語名を混ぜない
- `category` と `subtype` で棚を分ける
- 読者が辞書引きできるように関連語を 3〜5 つ残す
- 正式名称と会話での呼ばれ方が違う場合は両方書く
- 初出の略称は必ず展開する（例：CC ＝ Claude Code）
