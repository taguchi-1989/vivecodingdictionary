# Ponchi Quality Review Plan

## 目的

生成画像の品質を「なんとなく見る」状態から、基準画像との差分を見ながら判断できる状態にする。

この運用では、機械スコアは合否判定ではなく、外れ値を拾うための補助として使う。最終判断は人間が `OK` / `微修正` / `NG` を付ける。

## 最小ワークフロー

1. `assets/ponchi/final` などの画像フォルダをUIに読み込む。
2. 品質基準にしたい画像を1枚選ぶ。
3. UIが全画像を基準画像と比較してスコアを出す。
4. スコアが低いもの、警告が多いものから人間が確認する。
5. 各画像に `OK` / `微修正` / `NG` とメモを付ける。
6. CSVで結果を書き出し、再生成・修正キューに回す。

## 見る観点

- `色味`: 平均RGBと彩度が基準から離れていないか。
- `色制限`: 生成本文は `docs/ponchi_color_acceptance_gate.md` の本文パレットだけで成立しているか。黄、緑、赤、紫、茶、オレンジ、シアン/ティール、紫寄りの青、ブランド色はNG。
- `明るさ`: 全体が暗すぎる、薄すぎる、白飛びしていないか。
- `コントラスト`: 印刷・縮小時に読める強さがあるか。
- `線/密度`: エッジ密度が極端に低い、または細かすぎないか。
- `余白`: 被写体が小さすぎる、詰まりすぎる、中央から外れすぎていないか。
- `サムネイル`: 小さく並べた時に同じシリーズに見えるか。
- `意味`: 図鑑本文の概念を誤解させる小物や場面になっていないか。

## スコアの扱い

UIの機械スコアは100点満点で、以下を合成する。

- 色差
- 明度差
- 彩度差
- コントラスト差
- エッジ密度差
- 余白/被写体量差
- アスペクト比差

目安:

- `85-100`: まず問題なし。人間は軽く確認する。
- `70-84`: 微妙なズレがある。並びで確認する。
- `0-69`: 外れ値候補。優先的に確認する。

## 色ルール

ポンチ絵の本番画像は、シリーズ統一のため **生成本文を白・黒・グレー・指定青5色に限定する**。この節は運用説明であり、合格判定の正は `docs/ponchi_color_acceptance_gate.md` と `scripts/ponchi_color_audit.py` に置く。

許可:

- `#FFFFFF`
- `#F7F9FC`
- `#1A1A1A`
- `#6B7280`
- `#EAF1FB`
- `#D6E6FA`
- `#8DB7E8`
- `#3F7FD1`
- `#123E82`

アンチエイリアス、縮小、軽い影で近傍色が出ることはあるが、見た目として上記の白・黒・グレー・指定青から外れる色は使わない。

禁止:

- 黄、緑、赤、紫、茶、オレンジなどの差し色
- 虹色、カラフルなUI、暖色の強いハイライト
- シアン、ティール、紫寄りの青、任意の中間青
- ブランドカラーの再現
- ロゴや商標を連想させる固有色
- 写真サムネイル、製品UI、カラフルなグラフ

例外:

- 公式ロゴ、公式アイコン、公式マークを後合成したピクセルだけは元の色を保持してよい。
- 公式素材例外は `docs/brand_usage_audit.md` に出典とローカルパスが記録され、生成本文から分離できる場合に限る。
- 写真・スクリーンショット・製品UIを白黒青に寄せて使う運用はしない。意味が必要な場合は、形・位置・線の密度で抽象図解として描き直す。

## 基準画像の選び方

最初は1枚だけでよいが、運用が固まったらカテゴリごとに基準を分ける。

- サービス項目: `B-*`
- 人物・組織項目: `C-*`
- モデル項目: `D-*`
- 技術用語: `F-*`
- LLM概念: `G-*`
- 一般AI用語: `J-*`

基準画像は「一番うまい絵」ではなく、「今後の量産で守りたい平均的な正解」を選ぶ。

## 次に広げるなら

- 複数基準画像の平均スコア化。
- `ledgers/ponchi_generation_queue.csv` との突合。
- CSV結果から `needs_regen` キューを自動生成。
- CLIP embedding による意味・構図の類似度チェック。
- 章ごとの色味ばらつきレポート。
