# 本のように読むリーダー (book reader)

327 件のエントリを「もくじから選んで本のように読める」統合ポータル。HTML 1 枚で完結します。

## 使い方

`drafts/book/index.html` をブラウザでダブルクリック。`file://` で動きます（ローカルサーバ不要）。

## 構成

```
┌──────────────────────────────────────────┐
│ 上部ナビ：ホーム/検索/ルート/前付け/あとがき │
├──────────┬──────────────────────────────┤
│ もくじ    │ 本文表示エリア                 │
│          │                              │
│ A はじめに │  - ホーム: 4 つの入口カード     │
│  A-1 …   │  - エントリ選択: タイトル・読み │
│  A-2 …   │    タグライン・本文 ・関連語    │
│          │    ・前後ナビ                  │
│ B サービス │                              │
│  B-1 …   │                              │
│  …       │                              │
└──────────┴──────────────────────────────┘
```

## 機能

- **左サイドバー もくじ**: A〜J 章ごとにエントリを折りたたみ表示。クリックで本文表示
- **もくじフィルタ**: 入力欄でエントリを絞り込み（章を自動展開）
- **本文表示**: タイトル・読み・状態バッジ・タグライン・本文（Markdown → HTML 変換）・関連用語ピル
- **関連語ピル**: クリックで該当エントリへジャンプ
- **前後ナビ**: 同章内の前後エントリへワンクリック移動
- **URL ハッシュ**: 各エントリ表示時に `#A-1` のように URL に反映 → リロード後も同じ場所に戻れる
- **上部ナビ**: 既存の検索 UI / 読者ルート / 前付け / あとがき への横移動

## データ

`drafts/search/index.json` と同じインデックスを使用。エントリの本文（誌面に出る部分）も含む。

`scripts/build_search_index.py` が以下を行います：

1. `content/entries/**/*.md` を全件スキャン
2. YAML フロントマター・tagline・本文（裏台帳メモを除く）を抽出
3. JSON にして `drafts/search/index.json` に書き出し
4. `drafts/search/index.html` と `drafts/book/index.html` の `<script id="searchData">` タグ内に直接埋め込む

## 4 つの HTML ビューの関係

| ファイル | 役割 |
|:--|:--|
| `drafts/book/index.html` | 本のように読む（このリーダー） |
| `drafts/search/index.html` | 327 件を検索する |
| `drafts/reading_routes/index.html` | 4 つの読者タイプ別おすすめルート |
| `drafts/front_section/index.html` | 前付け 7 見開きと巻末あとがきへの導線 |

ホームから他のビューに飛べます（上部ナビ）。
