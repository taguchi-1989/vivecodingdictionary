# 確認用レビューシート

## 確認してほしい順番
1. まず、読者に合っているか
2. 次に、本文量が多すぎないか
3. 次に、デザイン案の方向性が合うか
4. 最後に、細かい言い回しを見る

## 本文テンプレについて
現時点の共通本文:

1. ひとことで
2. 何をしてくれるか
3. 実務での意味
4. 非エンジニア視点のつまずき
5. 最初にどこまで理解すればよいか
6. 私のコメント
7. コミュニティ補完メモ
8. 出典メモ
9. 備考

確認ポイント:
- この順番で読みやすいか
- `私のコメント` は必要か
- `コミュニティ補完メモ` は公開ページに出すか、裏台帳だけにするか
- `出典メモ` は公開ページに出すか、裏台帳だけにするか

## デザイン案について

### A: 図鑑カード型
合う項目:
- Gemini
- JavaScript
- React

見る点:
- 図鑑らしさがあるか
- 情報が詰まりすぎないか
- ぱっと引けるか

### B: 比較マップ型
合う項目:
- Geminiモデル比較
- ベンチマーク比較

見る点:
- 違いが直感的に見えるか
- スペック表に寄りすぎないか
- 更新負荷が高すぎないか

### C: Before / After 型
合う項目:
- TypeScript
- ESLint

見る点:
- 何が嬉しいかが伝わるか
- 単純化しすぎていないか
- 非エンジニア向けの入口として自然か

### D: ワークフロー型
合う項目:
- Next.js
- CI/CD
- GitHub Actions

見る点:
- 流れが見えるか
- 横長すぎないか
- 工程説明として使えるか
