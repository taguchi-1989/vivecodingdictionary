# B-5 GitHub Copilot ponchi regeneration prompt

## Purpose

Generate the base illustration for the B-5 GitHub Copilot entry. The final
image is a 2:1 WebP for the book spread.

## Key idea

GitHub Copilot is an AI code-completion extension added to an existing editor.
The image should make the before/after difference obvious: without Copilot the
developer is stuck typing boilerplate; with Copilot a pale suggestion appears
and the developer can accept it with Tab.

## Base image prompt

Create a 2:1 landscape editorial spot illustration for a Japanese
AI/development glossary entry about GitHub Copilot.

Canvas: 1254x627, white paper background.

Composition:
- Use a clear left/right before-after structure.
- Left side: a developer at a laptop, paused with hands above the keyboard,
  looking at a plain editor window with an empty cursor line.
- Right side: the same developer posture or a matching laptop/editor scene,
  but a pale ghost suggestion appears as simple horizontal gray strokes, and a
  single Tab key icon is being pressed.
- Put a narrow vertical divider or page-fold line between the two halves.
- Reserve a quiet, completely blank white clearspace area in the upper right
  for a later official logo overlay. Do not draw a box, border, badge, label,
  icon, or placeholder around this area. Leave enough clearspace for a large
  official lockup about one third of the canvas width.

Character continuity:
- Use recurring Character B from `docs/ponchi_character_bible.md` for the
  developer/explainer: anonymous adult man, short black hair, minimal dot eyes,
  light gray plain long-sleeve shirt, dark simple pants, no glasses, no beard,
  and no logo clothing.
- Do not introduce a new mascot or brand character.

Style:
- Clean simple editorial line illustration, not pencil sketch.
- Use smooth uniform black lines, flat light gray fills, minimal shading, no
  hatching, no scratchy hand-drawn texture, no marker texture, and no painterly
  look.
- The image should feel like a clear technical-book diagram with friendly human
  figures, not a rough drawing.
- Human-centered but not cute or mascot-like.
- Follow `docs/ponchi_image_generation_rules.md`: pure white background
  `#FFFFFF`, optional off-white `#F7F9FC`, black linework `#1A1A1A`, neutral
  gray secondary strokes `#6B7280`, and only the approved blue palette:
  `#EAF1FB`, `#D6E6FA`, `#8DB7E8`, `#3F7FD1`, `#123E82`.

Blue usage:
- Use `#EAF1FB` or `#D6E6FA` only for the active completion suggestion on the
  right side.
- Use `#8DB7E8` only for the accepted path or small glow ticks.
- Use `#3F7FD1` only as a tiny active cursor/key accent if needed.
- Use `#123E82` only for rare structural emphasis, and avoid it if black/gray
  works.
- Do not scatter blue sparkles, random highlights, clouds, or decorative
  strokes. Do not use a blue background, large blue filled areas, teal,
  purple-blue, cyan, or brand blue.
- Keep the left side almost monochrome so the before/after contrast is clear.

Text and UI:
- Do not include readable text, product UI screenshots, code snippets,
  Japanese text, English labels, brand marks, trademarks, or watermarks.
- Any editor content must be abstract lines only.
- The Tab key can be shown as a simple key shape with no readable label if text
  would be distorted.

Logo rule:
- Do not generate, imitate, redraw, or approximate any GitHub or Copilot logo.
- Do not generate, imitate, redraw, or approximate the Copilot icon, Octocat,
  GitHub marks, official app icons, official mascots, brand-color substitutes,
  or real product UI.
- The official logo will be composited afterward from local official assets.

Avoid:
- Photorealism, 3D render, glossy icon style, mascot character, dense dashboard,
  fake terminal text, fake browser chrome, speech bubbles, heavy blue coloring,
  bordered logo panel, and advertisement-like layout.

## Post-processing

- Overlay an official GitHub Copilot logo asset after generation.
- Preferred local logo asset:
  `assets/logos/github/GitHub_Logos/GitHub Logos/PNG/GitHub_Copilot_Lockup_Black_Clearspace.png`
- Place it in the reserved upper-right clearspace without a surrounding box.
- Target logo size: about `500-520px` wide on a `1254x627` canvas, large enough
  to identify at book preview size while staying inside the upper-right
  clearspace.
- Keep the logo's original aspect ratio and clearspace.
- Do not recolor, stretch, crop, redraw, or distort the logo.
