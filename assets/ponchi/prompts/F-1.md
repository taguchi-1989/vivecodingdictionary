# F-1 JavaScript ponchi regeneration prompt

## Purpose

Generate a 2:1 book ponchi image for the JavaScript entry.

## Key idea

JavaScript is the language that makes web pages move, and it also connects to a
larger ecosystem: TypeScript, UI libraries, browsers, and server runtimes.

## Base image prompt

Create a 2:1 landscape editorial spot illustration for a Japanese
AI/development glossary entry about JavaScript. Canvas 1254x627, pure white
background.

Composition:
- Center: a generic language core node shown as a simple code-file tile with
  abstract brackets and dots, not a real JS logo and no readable letters.
- Top path: three stacked generic building blocks representing type layer,
  UI layer, and framework layer. Use abstract icons only, no text labels.
- Bottom path: split into two simple environments: browser window icon on the
  left and server box icon on the right.
- Add one reader figure looking at the structure from the side, as if asking
  where the language runs.
- Use clear arrows and generous whitespace. No dense diagram.

Character continuity:
- Use the recurring Character A reader/listener from
  `docs/ponchi_character_bible.md`: anonymous adult woman, shoulder-length
  black hair, minimal dot eyes, white or very light gray top with a dark simple
  cardigan outline, no accessories, no patterned clothing.
- Do not introduce a new character design.
- No robot is needed for this JavaScript image unless it is a tiny neutral
  helper shape in the background; the language ecosystem should remain the
  subject.

Style:
- Clean simple editorial line illustration, not pencil sketch.
- Smooth uniform black lines, flat light gray fills, minimal shading.
- No hatching, scratchy hand-drawn texture, marker texture, or painterly look.

Color:
- Follow `docs/ponchi_image_generation_rules.md`.
- Use only `#FFFFFF`, `#F7F9FC`, `#1A1A1A`, `#6B7280`, `#EAF1FB`, `#D6E6FA`,
  `#8DB7E8`, `#3F7FD1`, `#123E82`.
- Use blue only for the active connection paths from the central code tile.

Text and logo rules:
- No readable text, no code snippets, no product UI screenshots, no logos, no
  trademarks, no brand colors, no watermarks.
- Do not generate JS, TypeScript, React, Next.js, browser, or Node.js logos.
- Do not generate official app icons, product icons, official marks, mascots, or
  brand-color substitutes.
- Generic browser and server icons are allowed only if they stay abstract and
  cannot be mistaken for real brand assets.

Avoid:
- Yellow JavaScript logo styling, colorful framework logos, dense dashboard,
  dark background, speech bubbles, decorative blue sparkles.
