# F-60 GitHub ponchi regeneration prompt

## Purpose

Generate a 2:1 book ponchi image for the GitHub entry. Official GitHub logo is
composited after generation.

## Key idea

GitHub is a shared cloud home for a repository, with issues, pull requests,
automation, and review activity gathered around it.

## Base image prompt

Create a 2:1 landscape editorial spot illustration for a Japanese
AI/development glossary entry about GitHub. Canvas 1254x627, pure white
background.

Composition:
- Center: a generic repository hub shown as a clean folder/cloud hybrid icon,
  with no logo and no readable text.
- Around it: four simple surrounding function panels represented by icons only:
  task checklist, merge/review branch, gear/check automation, and AI assistance
  sparkle. No labels.
- Add two or three small developer figures around the hub: one sending an
  upward arrow, one reviewing a panel, one watching an automated check.
- Reserve a clean blank white clearspace area in the upper right for the
  official GitHub logo overlay, large enough for a 500-520px wide lockup.
- Do not draw anything in the logo clearspace.

Character continuity:
- Use recurring Character B from `docs/ponchi_character_bible.md` for developer
  figures: anonymous adult men with short black hair, minimal dot eyes, light
  gray plain long-sleeve shirts, dark simple pants, no glasses, no beards, no
  logo clothing.
- If AI assistance is shown, use only a tiny recurring Character C pet robot:
  rounded-rectangle head, two dot eyes, small box body, no chest mark, no text.

Style:
- Clean simple editorial line illustration, not pencil sketch.
- Smooth uniform black lines, flat light gray fills, minimal shading.
- No hatching, scratchy hand-drawn texture, marker texture, or painterly look.

Color:
- Follow `docs/ponchi_image_generation_rules.md`.
- Use only `#FFFFFF`, `#F7F9FC`, `#1A1A1A`, `#6B7280`, `#EAF1FB`, `#D6E6FA`,
  `#8DB7E8`, `#3F7FD1`, `#123E82`.
- Use blue only for active collaboration paths and check highlights.

Text and logo rules:
- No readable text, no product UI screenshots, no code snippets, no generated
  logos, no trademarks, no brand colors, no watermarks.
- Do not generate, imitate, redraw, or approximate the GitHub logo or Octocat.
- Do not generate, imitate, redraw, or approximate official app icons, product
  icons, official marks, official mascots, brand-color substitutes, or real
  GitHub UI.
- The official GitHub logo will be composited afterward from local official
  assets.

Avoid:
- Dark GitHub UI, fake web app screenshots, speech bubbles, mascot character,
  decorative blue sparkles, bordered logo panel, advertisement-like layout.
