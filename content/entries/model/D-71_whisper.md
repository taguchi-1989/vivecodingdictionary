# Whisper

## tagline

OpenAI が公開した音声認識モデルです。99 言語に対応し、Apache 2.0 ライセンスで無償利用できます。

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## 何をしてくれるか

音声ファイルをテキストに変換します。日本語を含む 99 言語の文字起こしと翻訳に対応し、ローカル環境でも動かせます。派生ライブラリ whisper.cpp（ウィスパーシーピーピー）を使うと CPU だけで軽量推論できます。

## どこで出会うか

会議録や動画の字幕生成を自動化したいときに名前が出ます。Aqua Voice（B-18）や Amical（D-70）など音声系サービスの内部でも採用されており、API 経由で自前アプリに組み込む場面でも登場します。

## メイン図

### 図の狙い

音声が入力されて文字起こし結果が出力されるまでの流れを一枚で示します。多言語対応とローカル推論の選択肢があることも伝えます。

### A. ワークフロー（figure_type: workflow）

- 入力: マイク録音・音声ファイル（mp3 / wav 等）
- 処理: Whisper モデルが音声をデコード（クラウド API またはローカル実行）
- 出力: テキスト（文字起こし）または翻訳テキスト
- 分岐: クラウド API 利用 ／ whisper.cpp でローカル実行

## この用語の見どころ

### 1. 役割

音声をテキストへ変換するオープンな基盤モデルです。

### 2. うれしさ

無償・ローカル動作可能で、商用利用も許可されています。

### 3. 注意点

精度は音質や話速によって変わることがあります。

### 4. どこで役立つか

議事録自動化や動画字幕生成など音声テキスト化の場面。

### 5. はじめに

API 利用とローカル実行（whisper.cpp）の 2 択を把握します。

### 6. 深掘り先

OpenAI API、whisper.cpp、Dictation（G-3）。

## 開発フローでの位置（必須）

1. 音声ファイルを準備する — マイク録音・既存動画から音声を抽出
2. 実行方法を選ぶ — OpenAI API 呼び出し、またはローカルで whisper.cpp
3. モデルに渡す — ファイルと言語コードを指定してリクエスト送信
4. テキストを受け取る — 文字起こし結果を議事録・字幕・データ処理に活用

## コミュニティ補完メモ

- Dictation（G-3）は「音声入力で文字を書く」という汎用行為の概念エントリ。Whisper は OpenAI が公開した具体的な実装モデルで、Dictation の基盤になる存在として住み分けします。
- Aqua Voice（B-18）・Amical（D-70）は Whisper を内部で利用するサービスエントリ。Whisper 本体の詳細は本エントリに集約します。
- OpenAI（C-1）との関係：Whisper は OpenAI が公開したモデルの一つで、GPT 系とは別に音声特化で独立しています。
