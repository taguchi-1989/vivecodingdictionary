# gpt-oss

## tagline

OpenAI 初の公式オープンウェイトモデルです。20B と 120B の 2 サイズを提供します。

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## 何をしてくれるか

OpenAI が 2025-08 に Apache 2.0 で公開したモデルです。20B・120B の 2 サイズがあり、手元で推論できます。

## どこで出会うか

Hugging Face の OpenAI 公式ページでウェイトを配布しています。「クラウド API を使わず GPT 系品質を試したい」記事や OpenAI の方針転換ニュースで名前が出ます。

## メイン図

### 図の狙い

「OpenAI が初めてウェイトを公開した」という構造的な転換と、20B / 120B の 2 サイズの使い分けを 1 枚に収めます。

### B. 登場シーン（figure_type: structure）

- シーン1: 研究者がローカルサーバーに 20B をダウンロードして推論コストを抑える
- シーン2: 企業が 120B を社内 GPU に載せて外部 API を使わずに GPT 系品質を確保する
- シーン3: OSS 開発者が Apache 2.0 の権限でファインチューニング済みモデルを公開する
- 並べる基準: ウェイトを手元で動かす 3 つの利用文脈を用途別に並置

## 会話での使い方例

「gpt-oss の 20B なら Hugging Face からダウンロードして手元で動かせますよ。」

## この用語の見どころ

### 1. 役割

OpenAI 製のウェイト公開モデルです。手元での推論と改造が可能です。

### 2. うれしさ

Apache 2.0 ライセンスで商用・改造が自由です。API コストも不要です。

### 3. 注意点

GPT-4o 系のクローズドモデルより性能は下で、GPU 環境が必要です。

### 4. どこで役立つか

外部にデータを出せない環境や、コスト削減のローカル推論に向きます。

### 5. はじめに

「OpenAI 初のオープンウェイト」「Apache 2.0」「20B / 120B」の 3 点が要点です。

### 6. 深掘り先

Llama 系、DeepSeek、ファインチューニング

## 開発フローでの位置（必須）

1. サイズを選ぶ — 軽量用途は 20B、品質重視は 120B で GPU メモリを確認
2. ウェイトを取得 — Hugging Face の OpenAI 公式ページからダウンロード
3. ローカルで起動 — 推論ランタイムに載せて動作を確認する
4. 用途に合わせる — 必要に応じてファインチューニングや量子化を施す
5. API サーバー化 — 他ツールから呼び出せる形に整えてアプリに組み込む
