# DeepSeek R1

## tagline

強化学習（RL）で推論能力を獲得した、オープンウェイトの Reasoning モデルです。

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## 何をしてくれるか

強化学習を重ねることで、数学・コーディング・論理問題での推論能力を高めたモデルです。重みはオープンウェイトで公開されており、研究者や企業が手元の環境で動かせます。

## どこで出会うか

2025 年 1 月、o1 系と同等の推論性能をオープンウェイトで達成し「DeepSeek ショック」と呼ばれました。deepseek.com の API や Hugging Face で利用できます。

## メイン図

### 図の狙い

o1 系（クローズド・高価）と R1（オープンウェイト・低コスト）を横並びに比較し、同等の推論能力を異なるコスト構造で実現している点を示します。

### B. 登場シーン（figure_type: comparison）

- シーン1: 研究者が Hugging Face から R1 の重みを取得し、手元のサーバーで動かす
- シーン2: 企業が API コストを抑えるため deepseek.com の API 経由で推論タスクを処理する
- シーン3: ニュースで「o1 と同等の性能」と紹介され、非エンジニアが名前を目にする
- 並べる基準: オープン vs クローズドのコスト・利用形態の対比

## 会話での使い方例

「R1 はオープンウェイトで o1 系と近い推論ができるので、コスト比較に使えますよ。」

## この用語の見どころ

### 1. 役割

強化学習で推論能力を伸ばした DeepSeek のオープンウェイト推論モデルです。

### 2. うれしさ

o1 系に近い推論精度を、オープンな重みで利用できます。

### 3. 注意点

重みの運用にはサーバー環境が要り、個人利用の敷居は高めです。

### 4. どこで役立つか

数学・論理・コード検証など、段階的思考が必要なタスク。

### 5. はじめに

API で試せる点とオープンウェイトで入手できる点の二面性を把握します。

### 6. 深掘り先

DeepSeek V3、o1 系、強化学習

## 開発フローでの位置（必須）

1. タスク難易度を見る — 数学・論理・深いコード推論が要るか判断する
2. モデルを選ぶ — o1 系との性能・コストを比較し R1 を候補に入れる
3. アクセス方法を決める — API 経由か重み取得（Hugging Face）かを選ぶ
4. 推論結果を評価する — 応答の思考ステップと精度を確認し採否を決める
