# Claude 4 系

## tagline

Claude シリーズの第 4 世代モデル群です。4.0 → 4.1 → 4.5 → 4.6 → 4.7（現行）と版を重ねてきました。

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## ひとことで

Claude 4 系は、Anthropic の Claude シリーズの 4 世代目です。3 系（Opus／Sonnet／Haiku）の設計思想を受け継ぎつつ、コード・推論・長文処理で前世代より強くなっています。

## 何をしてくれるか

従来の Claude と同じく、文章生成・コード生成・画像理解・ツール呼び出しを担います。4 系ではツール呼び出しの安定性と長時間タスクの持続力が伸び、Claude Code などエージェント用途での主力モデル帯になりました。

ラインナップは従来と同じ 3 段階です。

- **Opus（最も賢いモデル）**：深く長く考える仕事（難しい調査、複雑な設計、長いコード理解）
- **Sonnet（中間のモデル）**：毎日のコーディング・文章・要約で最初に選ぶ主力
- **Haiku（アンカーモデル）**：軽量・高速。大量の短いタスク、低コスト運用

世代内のマイナー版は **4.0 → 4.1 → 4.5 → 4.6 → 4.7** と進んできました。現行は 4.7 です。

## バイブコーディングでの位置づけ

Claude Code の主力モデル帯です。「どの版のどれを使うか」で、コスト・速度・品質のバランスが変わります。

特に **4.5 で体感が大きく変わった**という声が多く、本書の著者感覚でも「ここでシンギュラリティを感じた」と言える跳躍点がありました。以降の 4.6・4.7 はその安定化と能力の伸長です。

Pro／Max プランの範囲内で自動選択される場面と、設定で明示する場面があります。多用するタスクがどの版で回っているかを把握しておくと、請求や体感品質の変化に気づきやすくなります。

## この用語の見どころ

### 1. 役割

Claude シリーズ第 4 世代です。3 段階の賢さでタスクと予算を分けます。

### 2. うれしさ

仕事の重さに合わせて最も賢い／中間／アンカーの 3 段階を選べます。

### 3. 注意点

版（4.0〜4.7）によって性能と価格が動くので、評価日を意識して読みます。

### 4. どこで役立つか

コーディング、長文読解、複雑な指示のあるタスク全般。

### 5. 最初に理解する範囲

Opus／Sonnet／Haiku の役割分担と、「4.5 が転換点」「現行は 4.7」という版の流れ。

### 6. 深掘り先

Extended Thinking、Claude 4.5 系の詳細、Sonnet 3.5 との差分、Claude Code での自動選択。

## 開発フローでの位置（必須）

1. タスクの重さを見積もる — 調査系／実装系／反復系
2. 段階を選ぶ — 最も賢い（Opus）／中間（Sonnet）／アンカー（Haiku）
3. 実行 — Claude Code、Claude.ai、API のいずれか経由
4. 結果と費用を評価 — 必要なら一段上／下に切り替える
