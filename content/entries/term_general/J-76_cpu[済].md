# CPU

## tagline

Central Processing Unit の略。コンピュータの汎用処理を担う頭脳チップです。

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## 何をしてくれるか

プログラムの命令を1つずつ順番に読み取り、計算・判断・制御を行います。並列度は低いですが、複雑な分岐処理が得意です。Intel Core、AMD Ryzen、Apple M シリーズなど製品ごとに性能が異なります。

## どこで出会うか

ローカル LLM（大規模言語モデル）を自分のパソコンで動かす記事で「CPU 推論」として出会います。llama.cpp などで軽量モデルを動かす際、GPU なしで動くかが話題になります。

## メイン図

### 図の狙い

CPU と GPU の役割分担を「指揮者と作業員集団」の対比で示し、どちらが何を担うかを直感的に掴んでもらう。

### B. 登場シーン（figure_type: comparison）

- シーン1: CPU — 指揮者。1人で複雑な判断・分岐をこなす
- シーン2: GPU — 大勢の作業員。同じ演算を大量並列でこなす
- シーン3: ローカル LLM — 軽量モデルなら CPU だけでも動作可能
- 並べる基準: 並列度と得意処理の違いによる役割対比


## 会話での使い方例

「軽量 LLM は CPU だけでも動かせて、API コストを節約できます。」

## この用語の見どころ

### 1. 役割

汎用命令を順次実行するコンピュータの制御中枢です。

### 2. うれしさ

GPU なしでも軽量 LLM をローカル動作させられます。

### 3. 注意点

並列演算が苦手なため、大規模 LLM 推論には不向きです。

### 4. どこで役立つか

ローカル AI ツールの動作要件確認や環境選定の場面です。

### 5. はじめに

CPU と GPU の役割分担の違いを把握することが出発点です。

### 6. 深掘り先

GPU、RAM、Apple Silicon

## 開発フローでの位置（必須）

1. 環境選定 — 手元マシンの CPU スペックを確認する
2. モデル選定 — CPU 推論対応の軽量モデルを選ぶ
3. ツール導入 — llama.cpp 等の CPU 推論エンジンをセットアップする
4. 動作確認 — CPU 使用率とレスポンス速度を確かめる
5. 本番比較 — GPU 環境や API と速度・コストを比べて判断する

## コミュニティ補完メモ

- GPU（J-77）との住み分け: CPU は「汎用・分岐に強い・直列処理」、GPU は「専用・並列に強い・繰り返し演算」。LLM 推論の主役は GPU だが、軽量モデルの CPU 動作は CPU エントリで扱う
- RAM（J-71）との住み分け: RAM はメモリ（作業台）、CPU は演算装置（頭脳）。ローカル LLM では両方のスペックが問われるため相互参照
- H100（J-72）との住み分け: H100 は GPU の高性能品種。大規模 LLM には GPU が必要というコンテキストで CPU との対比が自然に出る
- Apple Silicon（関連）: M シリーズは CPU と GPU が統合メモリを共有する設計のため「ローカル AI の入口」として別途言及価値がある
