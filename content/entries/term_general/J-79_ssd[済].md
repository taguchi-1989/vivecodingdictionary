# SSD

## tagline

Solid State Drive の略。フラッシュメモリに保存する永続ストレージで HDD より高速・静音です。

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## 何をしてくれるか

OS・アプリ・データを電源を切っても保存します。SATA 接続で HDD の 5〜10 倍、NVMe では 30〜50 倍以上の読み書き速度が出ます。

## どこで出会うか

PC 購入時のスペック表やローカル AI ツールの推奨環境で目にします。ollama でローカル LLM を切り替える際、SSD であればモデルのロード待ちがほぼ消えます。

## メイン図

### 図の狙い

HDD と SSD の速度差を「待ち時間の長さ」で直感的に対比する。

### A. Before / After（figure_type: before_after）

- Before
  - 状況: HDD 搭載 PC でローカル LLM をロード
  - 視覚要素: プログレスバーが長く伸びている
  - つまずき: モデル切り替えのたびに数十秒待つ
- After
  - 状況: NVMe SSD に換装してロード
  - 視覚要素: プログレスバーが一瞬で完了
  - うれしさ: 待ち時間がほぼ消え、作業リズムが途切れない

## 会話での使い方例

「NVMe SSD に換装したら、Llama のロードが体感で 5 倍速くなりました。」

## この用語の見どころ

### 1. 役割

PC のデータを高速に永続保存するストレージです。

### 2. うれしさ

AI モデルのロード時間が大幅に短縮されます。

### 3. 注意点

書き込み寿命（TBW）があり、頻繁な大容量書き込みで劣化します。

### 4. どこで役立つか

ローカル LLM の切り替えや大規模データ処理で差が出ます。

### 5. はじめに

SSD は方式、SATA / NVMe は接続規格、M.2 は形状と階層で区別します。

### 6. 深掘り先

HDD、NVMe、M.2

## 開発フローでの位置（必須）

1. 環境選定 — PC や開発サーバーのストレージ種別を SSD / HDD で確認します
2. モデル配置 — SSD 上に LLM モデルファイルを置き、ollama から参照します
3. ロード実行 — NVMe SSD なら数秒以内でモデルが展開されます
4. 切り替え — 別モデルへの切り替えも待ち時間なく繰り返せます

## 非エンジニアのつまずき

- SSD と一括りに言っても、SATA・NVMe・M.2 など複数の規格があり、アーキテクチャだけでは性能を判断しきれない難しさがあります
- HDD に比べて値段が高くなりがちで、特に大容量帯では価格差が広がります

## コミュニティ補完メモ

- HDD（J-78）との住み分け：J-78 は磁気ディスク方式の仕組みと歴史を担当。J-79（SSD）は HDD と対比しつつ「速度とバイブコーディングへの恩恵」に集中する
- SATA（J-80）との住み分け：J-80 は接続インターフェース規格の説明を担当。SSD の文脈では「接続規格は SATA / NVMe、形状は M.2、方式は SSD」と階層整理する
- M.2（J-81）との住み分け：J-81 は形状・スロット規格の説明を担当。本エントリでは M.2 を形状の一例として一言だけ触れ、詳細は J-81 に委ねる
