# パラメータ数の単位

## tagline

B＝Billion（十億）、T＝Trillion（兆）の略。LLM の重みの個数で容量感を示す単位です。

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## 何をしてくれるか

パラメータ数はニューラルネット内の学習可能な重みの総数です。数が多いほど知識の収容量が増え、複雑な推論にも対応しやすくなります。ただし推論コストと所要時間も概ね比例して増えます。

## どこで出会うか

モデル選定の場面で登場します。「8B ならノート PC で動く」「70B は GPU が要る」「400B 級はクラウド前提」という桁感があると判断が速くなります。MoE 構造では活性化パラメータが実態に近い数字です。

## メイン図

### 図の狙い

規模ごとのモデルと動作環境の対応を一目で示し、桁感で覚えるきっかけを作る。

### B. 登場シーン（figure_type: comparison）

- シーン1: 8B クラス — ノート PC でも動作可能（Llama 3.1 8B など）
- シーン2: 70B クラス — GPU サーバや高性能ワークステーションが必要
- シーン3: 400B 超クラス — クラウド API 経由での利用が現実的
- 並べる基準: モデル規模（パラメータ数）と必要ハードウェアの対応

## 会話での使い方例

「Llama 3.1 の 8B なら手元で動くので、開発時はそれで試して本番だけ 70B にします。」

## この用語の見どころ

### 1. 役割

LLM の容量感を「重みの個数」という数値で示す単位です。

### 2. うれしさ

桁を見るだけで動作環境の目安がわかり、モデル選定が速くなります。

### 3. 注意点

MoE モデルは総パラメータと活性化パラメータが大きく異なります。

### 4. どこで役立つか

ローカル実行とクラウド利用の切り分けを判断する場面に役立ちます。

### 5. はじめに

M（百万）→ B（十億）→ T（兆）と 3 桁刻みで増えると押さえましょう。

### 6. 深掘り先

MoE、量子化、VRAM

## 開発フローでの位置（必須）

1. 要件確認 — タスクに必要な推論精度を大まかに見積もります
2. モデル候補選定 — パラメータ数の桁でローカル・クラウドを仕分けます
3. 検証実行 — 小さいモデルで動作を確認し、精度不足なら上位に切り替えます
4. 本番決定 — コストと精度のバランスで最終モデルを決定します

## 備考

- GPT-4 のパラメータ数は OpenAI 公式未公表。「1.7T 超」は外部推定値（複数ソース）のため、「公称 1.7T 超とされる（推定値）」と留保をつける
- 量子化を適用するとモデルのファイルサイズは変わるがパラメータ数自体は変わらない（混同注意）
- パラメータ数と性能は必ずしも比例しない。小さくてもファインチューニング済みモデルが特定タスクで上回るケースがある
