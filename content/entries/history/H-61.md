# Preview 版という文化

## tagline

完成を待たず小出しにリリースする、AI 業界の標準慣行です。

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## 何をしてくれるか

AI モデルの更新サイクルは速く、「Preview」「Beta」「Experimental」などのラベルで未完成のまま先行提供し、フィードバックを得ながら育てる方法が業界全体に定着しています。

## どこで出会うか

公式ページや API ドキュメントの「Preview」表記が典型です。Gemini 2.0 Pro Preview のように 1 年以上ラベルが取れないケースもあり、「いつ GA になるか分からない」状況は珍しくありません。

## メイン図

### 図の狙い

Preview ラベルが乱立し、長期化する状況を時系列で示す。

### C. 概念図（figure_type: timeline）

- 中心に置く概念: 時系列の横軸と「Preview → GA」の矢印
- 周辺の要素: Preview / Beta / Experimental / Early Access の各ラベル、GA 到達まで 1 年以上のケース
- 関係の描き方: 横軸上に複数モデルが「Preview 停滞」している様子を並べる

## 会話での使い方例

「このモデルはまだ Experimental なので、仕様変動を前提に使うのが基本です。」

## この用語の見どころ

### 1. 役割

未完成のモデルを先行公開してフィードバックを集める慣行です。

### 2. うれしさ

早期に新機能を試せて、料金が抑えめな期間を活用できます。

### 3. 注意点

仕様が予告なく変わり、GA 時に料金が上がることがあります。

### 4. どこで役立つか

Preview 期間中に検証し、GA 後の本番移行可否を判断する場面に役立ちます。

### 5. はじめに

Preview は SLA なし・仕様変動あり、という前提から押さえます。

### 6. 深掘り先

GA、SLA、H-51 Preview から正式版への流れ

## 開発フローでの位置（必須）

1. リリース発表 — 「Preview」「Beta」など先行ラベルつきで公開される
2. 試用期間 — 料金抑えめ・SLA なしで触れる、フィードバックを送る
3. 仕様変動 — 予告なく API や料金が変わることがある
4. GA 移行 — 料金正式化・SLA 付与で本番利用できる状態になる
5. 長期 Preview — GA にならず 1 年以上 Preview のまま放置されることもある

## コミュニティ補完メモ

H-51「Preview から正式版への流れ」との住み分け：H-51 は Preview → GA の正常な移行フローと各段階の意味を扱う。H-61 は Preview が長期化する文化・慣行そのものを扱い、弊害（仕様変動・1 年放置）と業界特有の文化観察を前面に出す。GA の定義や SLA の説明は H-51 に任せ、H-61 では参照に留める。

Google（Gemini）が特に Preview 長期化が多いと揶揄される点は観察事実として備考に記す。誌面本文では特定企業への批判的記述を避け、事実として「1 年以上 Preview のケースもあります」と中立的に書く。
