# 拡張子早見表

## tagline

ファイル名末尾の `.js` `.md` `.yaml` などが「何のファイルか」を示す記号です。

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## 何をしてくれるか

拡張子（かくちょうし）はファイルの種類を一目で示します。`.ts` なら TypeScript（タイプスクリプト）コード、`.md` なら Markdown（マークダウン）文書、`.json` なら JSON（ジェイソン）形式の設定・データと読み取れます。AI が生成したファイルも必ずいずれかの拡張子を持ちます。

## どこで出会うか

VS Code（ブイエスコード）のファイルツリーや、AI との会話で「`index.ts` を修正しました」と表示されるときに見かけます。バイブコーディングでは AI が複数の拡張子をまとめて生成することがあり、「これは何?」と感じたときの辞書代わりになります。

## メイン図

### 図の狙い

代表的な拡張子をカテゴリ別に並べ、「種類 → 拡張子 → 一言メモ」の対応を一覧で見せます。

### B. 登場シーン（figure_type: structure）

- シーン1: Web コード — `.html`（骨組み）、`.css`（見た目）、`.js`（動き）
- シーン2: 型付きコード — `.ts`（TypeScript）
- シーン3: 設定・文書 — `.yaml`／`.json`（設定）、`.md`（文書）
- 並べる基準: ファイルの用途カテゴリ（コード／設定／文書／メディア）

## 会話での使い方例

「`.yaml` と `.json` はどちらも設定ファイルで、プロジェクトによって使い分けます。」

## この用語の見どころ

### 1. 役割

ファイルの種類をひと目で示す目印です。

### 2. うれしさ

「このファイルは何?」の疑問をすぐ解消できます。

### 3. 注意点

同じ `.js` でも役割は多様で、一覧だけで全部はわかりません。

### 4. どこで役立つか

AI 生成ファイルを受け取った直後の確認に使えます。

### 5. はじめに

`.md` `.json` `.yaml` の 3 種を先に覚えると使いやすいです。

### 6. 深掘り先

Markdown、YAML、JSON、TypeScript

## 開発フローでの位置（必須）

1. プロジェクト作成 — AI がフォルダと複数のファイルを一気に生成します
2. 拡張子を確認 — ファイルツリーを眺めて「何のファイルか」を把握します
3. 編集対象を絞る — `.ts` はコード、`.yaml` は設定と分けて考えます
4. AI に依頼 — 「この `.yaml` を修正して」と拡張子を交えて指示を出します
