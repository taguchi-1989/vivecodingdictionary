# VS Code 拡張機能

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## tagline

VS Code（F-30）に機能を追加するプラグインの総称です。素のエディタを自分の用途に育てる仕組みです。

## 何をしてくれるか

コード整形・Lint・AI 補完・テーマなど、開発に必要な機能をワンクリックで追加できます。インストールは「拡張機能」サイドバー（`Ctrl+Shift+X`）から検索するだけです。

## どこで出会うか

AI 開発を始めた際に「GitHub Copilot を入れてみよう」と案内されたとき、多くの人が初めて拡張機能を意識します。Prettier（整形）や ESLint（構文検査）の導入案内でも頻繁に名前を目にします。

## メイン図

### 図の狙い

VS Code の素の状態と、複数の拡張を入れた後の状態を人物視点で対比し、「育てる感覚」を伝える。

### B. 登場シーン（figure_type: structure）

- シーン1: 新しい言語を扱うとき — 言語サポート拡張を入れて補完が効くようにする
- シーン2: コードをチームで統一したいとき — Prettier / ESLint 拡張を入れて自動整形
- シーン3: AI 補完を使いたいとき — GitHub Copilot や Claude Code 拡張を追加する
- 並べる基準: 導入目的ごとのシーン（誰が・何のために）

## 会話での使い方例

「Prettier は VS Code 拡張機能で入れると保存時に自動整形されます。」

## この用語の見どころ

### 1. 役割

VS Code の機能をプラグイン単位で追加する仕組みです。

### 2. うれしさ

必要な機能だけ選んで入れられるため、軽量に保ちやすいです。

### 3. 注意点

入れすぎると起動が重くなり、信頼できないパブリッシャの拡張は危険です。

### 4. どこで役立つか

AI 補完・整形・Lint をまとめて整えるセットアップ時に役立ちます。

### 5. はじめに

Prettier と ESLint 拡張を入れるとセットアップの流れが体感できます。

### 6. 深掘り先

VS Code、Prettier、ESLint、GitHub Copilot

## 開発フローでの位置（必須）

1. 環境構築 — VS Code をインストールした直後に拡張機能のセットアップを行います
2. 拡張検索 — `Ctrl+Shift+X` でサイドバーを開き、用途ごとに検索します
3. インストール — ワンクリックで追加し、再起動なしで有効になります
4. カスタマイズ — settings.json で各拡張の動作を調整します
5. 保守 — 自動アップデートで挙動が変わることがあるため定期的に確認します

## 備考

- 拡張機能の数は 2026-04 時点で 5 万件超（Visual Studio Marketplace）。数が多いため「入れすぎ注意」は初心者向けに必ず触れる。
- 自動アップデートにより挙動が変わることがある。チーム開発では拡張バージョンを固定する `.vscode/extensions.json` の推奨リスト機能も存在する。
- Claude Code（B-7）の VS Code 拡張版は別途提供されており、エディタ内から AI エージェントを呼び出せる。
