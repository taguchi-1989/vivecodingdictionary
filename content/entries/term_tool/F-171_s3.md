# S3

## tagline

Amazon Simple Storage Service の略。AWS のオブジェクトストレージで、ファイルをバケット単位で管理します。

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## 何をしてくれるか

ファイル（オブジェクト）を「バケット」と呼ぶ入れ物に保存し、URL やキーで取り出せるストレージです。公称耐久性は 99.999999999%（イレブン・ナイン）で、事実上データ消失はありません。

## どこで出会うか

Claude や Cursor で生成したコードをデプロイするとき、画像・動画の配信先や AI 学習データの置き場として名前が出ます。AWS コンソールまたは CLI でバケットを作る手順が最初の接点です。

## メイン図

### 図の狙い

バケット／オブジェクト／キーの 3 層構造と、代表的な利用シーンを並べて見せる。

### C. 概念図（figure_type: structure）

- 中心に置く概念: バケット（Bucket）
- 周辺の要素: オブジェクト（ファイル＋メタデータ）、キー（パス）、ストレージクラス、静的サイト、AI 学習データ
- 関係の描き方: バケットを中心に、利用シーンを矢印で放射状に配置


## 会話での使い方例

「学習データは S3 のバケットに集約して、boto3 で読み込ませています。」

## この用語の見どころ

### 1. 役割

ファイルをバケットに置き、URLやAPIで読み書きするストレージです。

### 2. うれしさ

容量を気にせず使え、耐久性 11 nines で消失リスクがほぼゼロです。

### 3. 注意点

バケットの公開設定を誤ると情報漏洩になるため、デフォルトは非公開です。

### 4. どこで役立つか

静的サイト配信・AI 学習データ管理・ログ長期保管に幅広く使えます。

### 5. はじめに

バケット作成と IAM 権限の設定がまず理解すべき基礎です。

### 6. 深掘り先

IAM、ストレージクラス、AWS MCP


## 開発フローでの位置（必須）

1. バケット作成 — AWS コンソールまたは CLI でバケットと IAM ポリシーを用意します
2. データ投入 — boto3 や AWS CLI でオブジェクトをアップロードします
3. アクセス設計 — ブロックパブリックアクセスを確認し、必要な公開範囲を絞ります
4. 読み込み利用 — スクリプトや Lambda からキーを指定してオブジェクトを取得します

## コミュニティ補完メモ

- EC2（F-170）との住み分け：EC2 は「計算する」サーバー、S3 は「置く」ストレージ。セットで使うことが多い
- IAM（F-172）との関係：S3 バケットへのアクセス制御は IAM ポリシーで行うため、セットで理解が必要
- AWS MCP（I-50）との関係：Claude から S3 を操作するときに MCP 経由で呼ぶケースがある

## 備考

- 耐久性 99.999999999%（11 nines）は AWS 公称値。実際の SLA は異なる
- ストレージクラスは Standard / Standard-IA / Glacier 等があり、用途・コストで使い分ける
- 「ブロックパブリックアクセス」設定は 2023 年以降デフォルト ON が推奨。誤公開事故防止の要
