# a11y

## tagline

Accessibility（a と y の間に 11 文字あるため a11y）の略です。障害の有無や利用環境を問わず、誰でも Web を使えるよう設計・実装する取り組みです。


<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## 何をしてくれるか

画像への alt テキスト付与、セマンティック HTML（`<button>` や `<nav>` 等）の使用、キーボード操作への対応など、具体的な実装基準を示します。WCAG（Web Content Accessibility Guidelines）が A / AA / AAA の 3 段階で達成基準を定めています。

## どこで出会うか

Lighthouse（F-110）の Accessibility スコアや、Claude・Cursor が UI コードを生成した後のレビュー時に登場します。「コントラスト比が不足しています」「aria-label が必要です」といった指摘が a11y 対応の入口です。

## メイン図

### 図の狙い

a11y を気にしない UI と気にした UI を並べ、スクリーンリーダーやキーボード操作でどう変わるかを示す。

## 会話での使い方例

「a11y のチェックを Claude に頼んだら、alt 属性の漏れが大量に見つかりました。」

## この用語の見どころ

### 1. 役割

誰でも使える Web を実現するための設計・実装基準です。

### 2. うれしさ

AI 生成コードの品質を Lighthouse で自動スコア化できます。

### 3. 注意点

WCAG の AAA 準拠は工数が大きく、まず AA を目標にします。

### 4. どこで役立つか

UI 生成後のレビューや、公開前の品質チェックで役立ちます。

### 5. はじめに

alt テキストと WCAG AA の意味を押さえれば入門として十分です。

### 6. 深掘り先

WCAG、Lighthouse、ARIA

## 開発フローでの位置（必須）

1. UI 設計 — コントラスト比と操作順序を考慮してレイアウトを決める
2. コーディング — セマンティック HTML と alt テキストを AI に生成させる
3. 自動チェック — Lighthouse の Accessibility スコアで達成状況を確認する
4. 修正 — スコアの指摘（aria-label 不足等）を Claude に渡して修正させる
5. 公開 — AA 準拠を確認してリリースする

## コミュニティ補完メモ

- Lighthouse（F-110）との住み分け：Lighthouse は計測ツール全般を扱うエントリ。a11y はアクセシビリティの概念・基準を扱う。「Lighthouse の Accessibility スコアで a11y を測る」という関係
- HTML（F-4）との住み分け：HTML はマークアップ言語の基礎エントリ。a11y は HTML のセマンティクス活用を前提とした応用的な実装基準
- shadcn/ui（F-15）との住み分け：shadcn/ui はコンポーネントライブラリのエントリ。a11y 対応済みコンポーネントを提供している点での関連

## 備考

- WCAG 2.1 は 2018 年、2.2 は 2023 年公開。バージョンによって達成基準が異なるため、評価時は版を確認する
- 日本では JIS X 8341-3 として国内規格化されている（WCAG 2.1 準拠）
- prefers-reduced-motion（動きを減らすメディアクエリ）は前庭感覚障害への配慮で、近年の a11y 実装でよく話題になる
