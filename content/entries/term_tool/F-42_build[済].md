# ビルド

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## tagline

ソースコードを実行・配布できる形へ変換する一連の工程です。

## 何をしてくれるか

TypeScript を JavaScript へ変換し、複数ファイルをバンドル・圧縮する工程です。ブラウザは TypeScript を直接読めないためこの変換が必要です。

## どこで出会うか

AI に「`npm run build` を実行して」と言われたとき、または「ビルドが通らない」エラーで登場します。Vite や webpack などのツールが舞台裏でこの工程を担います。

## メイン図

### 図の狙い

ソースコード（人が書く形）と配布物（ブラウザが読む形）の間にビルドが入ることを示す。

### C. 概念図（figure_type: workflow）

- 中心に置く概念: ビルド工程
- 周辺の要素: TypeScript ファイル群、JavaScript 変換後ファイル、バンドル（1 ファイル化）、minify（圧縮）、出力フォルダ（dist/）
- 関係の描き方: 左から右への変換フロー矢印

## 会話での使い方例

「`npm run build` で TS のコンパイルと JS のバンドルがまとめて走ります。」

## この用語の見どころ

### 1. 役割

ソースコードを本番向けの配布物へ変換する工程です。

### 2. うれしさ

1 コマンドで変換・圧縮・最適化が同時に終わります。

### 3. 注意点

ビルドエラーは設定ファイルや依存関係の問題で起きることがあります。

### 4. どこで役立つか

本番デプロイ前や CI/CD パイプラインの中で必ず通ります。

### 5. はじめに

「ビルド＝人が書いたコードをブラウザ向けに変換すること」と掴むのが出発点です。

### 6. 深掘り先

Vite、webpack、TypeScript

## 開発フローでの位置（必須）

1. コード編集 — TypeScript・JSX・CSS などを書く
2. ビルド実行 — `npm run build` でビルドツールが起動する
3. 変換・バンドル — TS→JS 変換、ファイル結合、minify が走る
4. 出力確認 — `dist/` フォルダに成果物が生成される
5. デプロイ — 出力物をサーバーや CDN に配置する

## 非エンジニアのつまずき

- コンパイルとビルドがひとつながりの流れで進みますが、どこからどこまでがビルドなのかが最初は分かりにくく、「ビルドが失敗しました」のエラーが特に CI 環境では対処しにくかったです。
- ビルドにかかる時間も悩みの種でした。CI が走り始めてから静的解析＋ビルドで 8 分ほどかかることもあり、待ち時間がつらかったです。

## コミュニティ補完メモ

- Vite（F-41）はビルドツールの代表例。「ビルド」が工程の総称で、Vite はその実装の 1 つというスコープ分担。
- npm（F-40）との住み分け：`npm run build` は npm がコマンドを呼ぶ仕組みで、ビルド本体の処理は Vite 等が担う。
- テスト（F-43）は通常ビルドの前工程として CI に組み込まれる。
