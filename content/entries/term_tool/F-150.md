# MIT ライセンス

## tagline

著作権表示を残せば自由に使えるオープンソースライセンスです。商用利用・改変・再配布がすべて認められます。

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## 何をしてくれるか

ソフトウェアの著作権者が「著作権表示と本ライセンス文を残す条件で、自由に使ってよい」と宣言する仕組みです。商用製品への組み込みや改変・再配布が可能で、派生物のソース公開義務もありません。


## どこで出会うか

React・Vue・TypeScript・VS Code（F-30）・Node.js（F-80）など、広く使われる OSS（オープンソースソフトウェア）の多くが MIT です。Claude や Cursor が「このライブラリを使って」と提案する OSS の大半は MIT ライセンスで配布されています。


## メイン図

### 図の狙い

MIT ライセンスが「何を要求し、何を自由にするか」を一覧で示し、Apache 2.0 との違いを一目で比較できるようにします。

- シーン1: 開発者が OSS ライブラリを選ぶ場面で MIT か Apache かを見比べる

## この用語の見どころ

### 1. 役割

OSS の利用条件を定め、著作権保持者が自由使用を宣言します。

### 2. うれしさ

商用・改変・再配布がすべて可能で、使う側の制約が最小限です。

### 3. 注意点

特許権に関する条項がなく、特許リスクは自己判断で確認が必要です。

### 4. どこで役立つか

社内製品や商用 SaaS にライブラリを組み込む判断場面で役立ちます。

### 5. はじめに

著作権表示と本文を残す義務だけ覚えれば、利用判断の大半が出来ます。

### 6. 深掘り先

Apache 2.0、GPL、OSS


## 開発フローでの位置（必須）

1. ライブラリ選定 — OSS の README や npm でライセンス種別を確認します
2. 条件確認 — 著作権表示を残すだけでよいかを判断します
3. 組み込み — 商用製品・内部ツールへの取り込みを実施します
4. 表示対応 — ライセンス文を同梱または NOTICE ファイルに記載します
5. 法務確認 — 必要に応じて Apache 2.0 等との比較を法務に共有します

## コミュニティ補完メモ

- Apache 2.0（F-151）との住み分け：MIT は特許条項なし・短文で分かりやすい。Apache は特許権の明示的許諾があるため大企業が特許リスクを気にする場面では Apache を選ぶことがある。本エントリでは MIT の概要に集中し、Apache との差分は F-151 側で詳述する。
- GPL（F-152）との住み分け：GPL はコピーレフト（派生物もソース公開義務あり）。MIT との最大の違いはここで、F-152 側で詳述する。
- Creative Commons（F-153）との住み分け：CC はソフトウェア以外（文書・画像等）向け。本エントリはソフトウェアライセンスに限定する。

## 備考

- MIT ライセンス本文は約 200 単語と非常に短く、一般的に "permissive" ライセンスに分類されます。
- AS IS 条項：作者が「無保証」を明示しているため、利用者はソフトウェアの不具合リスクを自己負担する形になります。「ライセンスが緩すぎて何も保護されない？」という読者のつまずきは、AS IS 条項の説明で解消できます（作者が保護されない、という意味ではなく、ユーザーが保証を求められない、という意味）。
