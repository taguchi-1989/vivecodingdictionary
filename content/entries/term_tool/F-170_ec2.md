# EC2

## tagline

Amazon Elastic Compute Cloud の略。AWS（アマゾン ウェブ サービス）上で仮想マシンを従量課金で借りられるサービスです。

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## 何をしてくれるか

インスタンスと呼ぶ仮想マシンを数分で起動し、OS・CPU・メモリ・GPU を自由に選んで使えます。使った時間だけ課金される従量制で、不要になればすぐ削除できます。

## どこで出会うか

AI モデルの学習・推論を GPU インスタンス（p5 系：最大 8 枚の H100 搭載）で行う場面や、Claude Code でビルドしたアプリのホスティング先を選ぶ場面で名前が出てきます。AWS コンソールや構成図の中心にあるサービスです。

## メイン図

### 図の狙い

インスタンスタイプの種類と「止めても EBS は課金される」の構造を 1 枚で整理する。

### B. 登場シーン（figure_type: structure）

- シーン1: AI エンジニアが p5 インスタンスを起動して LLM（大規模言語モデル）の学習を回す
- シーン2: Web 開発者が T 系汎用インスタンスにアプリをデプロイして公開する
- シーン3: コスト担当者が Spot インスタンスで余剰枠を安く確保して処理を流す
- 並べる基準: 用途（GPU 学習 / アプリ公開 / コスト最適化）視点


## 会話での使い方例

「Claude が書いたモデルを EC2 の p5 インスタンスで学習させて、結果を S3 に置きました。」

## この用語の見どころ

### 1. 役割

AWS の仮想マシンサービスで、サーバーを秒単位で起動・停止できます。

### 2. うれしさ

物理サーバーを買わずに GPU クラスタも時間借りで使えます。

### 3. 注意点

停止中でも EBS（永続ディスク）の課金は続くため注意が要ります。

### 4. どこで役立つか

AI モデル学習・Web アプリ公開・バッチ処理など幅広い用途に使えます。

### 5. はじめに

インスタンスタイプの選び方と、停止と終了の違いが最初の理解ポイントです。

### 6. 深掘り先

S3、IAM、Reserved / Spot インスタンスの使い分け

## 開発フローでの位置（必須）

1. 要件整理 — 処理内容に合うインスタンスタイプ（CPU / GPU / メモリ）を選定します
2. 起動 — AWS コンソールまたは CLI でインスタンスを起動し、SSH で接続します
3. 実行 — 学習スクリプトやアプリを走らせ、成果物を S3 などに書き出します
4. 停止・終了 — 作業後はインスタンスを停止し、不要なら終了してコストを抑えます

## コミュニティ補完メモ

- B-23 AWS との住み分け：AWS はプラットフォーム全体、EC2 はその中の仮想マシンサービス。AWS エントリでは全体像を扱い、EC2 エントリではインスタンスタイプ・課金体系・ライフサイクルを掘り下げる
- F-171 S3 との住み分け：EC2 はコンピュート（処理）、S3 はストレージ（保存）。セットで登場する場面が多いが役割が異なる
- 停止（Stop）と終了（Terminate）の違い：停止はインスタンスを一時停止（EBS は保持・課金継続）、終了はインスタンスを削除（EBS も削除）。本文字数の関係で備考に詳細を記す

## 備考

- インスタンスタイプ体系（2026-04-30 時点）：汎用（M / T 系）/ 計算向け（C 系）/ メモリ最適（R 系）/ GPU（P / G 系：p5 = H100×8、p4d = A100 等）/ ストレージ最適（I / D 系）
- 停止中の課金詳細：コンピュート料金は停止中ゼロだが、EBS（Elastic Block Store）ボリュームはプロビジョン容量に対して課金が続く。読者つまずきの定番
- 課金モデル 4 種：オンデマンド（秒・時間単位）/ Reserved（1〜3 年予約で最大 72% 割引）/ Spot（余剰枠、最大 90% off・中断あり）/ Savings Plans（コミットメント割引）
- 公開年 2006 年：クラウドコンピューティングを世間に広めた初期サービスの 1 つ
