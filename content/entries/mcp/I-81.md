# MCP の登録・設定

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## tagline

MCP Server をクライアントに認識させる設定操作です。登録しないと使えません。

## 何をしてくれるか

MCP Server を Claude Code・Claude Desktop・Cursor などのクライアントに登録します。設定ファイルに名前・コマンド・環境変数を書けば、起動時に呼び出せます。

## どこで出会うか

自作やサードパーティの MCP Server を使うとき必ず通ります。Claude Code は `claude mcp add`、Claude Desktop は `claude_desktop_config.json` 編集、Cursor は `.cursor/mcp.json` を使います。

## メイン図

### 図の狙い

登録先ファイルの場所とクライアントの対応関係を 1 枚で示す。

### B. 登場シーン（figure_type: structure）

- シーン1: Claude Code — `claude mcp add <name> <command>` を実行、`~/.claude.json` に書き込まれる
- シーン2: Claude Desktop — `claude_desktop_config.json` の `mcpServers` キーを手編集
- シーン3: Cursor — `.cursor/mcp.json`（プロジェクト）または `~/.cursor/mcp.json`（ユーザー）
- 並べる基準: クライアント別の設定ファイルの場所

## 会話での使い方例

「Claude Code に MCP Server を登録するには `claude mcp add` で一発ですよね。」

## この用語の見どころ

### 1. 役割

クライアントが MCP Server を起動・接続できるよう登録する操作です。

### 2. うれしさ

一度登録すれば再起動後も自動で接続されます。

### 3. 注意点

クライアントごとに設定ファイルの場所と書式が異なります。

### 4. どこで役立つか

チームで共有するなら project スコープ、自分専用なら user スコープを使います。

### 5. はじめに

CLI か設定ファイル編集のどちらかを覚えれば登録できます。

### 6. 深掘り先

MCP Server、MCP Client、stdio／SSE トランスポート

## 開発フローでの位置（必須）

1. MCP Server を用意する — 自作または npm パッケージ等で取得
2. クライアントへ登録する — CLI コマンドまたは設定ファイルを編集
3. 環境変数を渡す — API キー等を `env` キーか OS の環境変数で設定
4. クライアントを再起動する — 設定を読み込ませて接続確認
5. ツールを呼び出す — プロンプトや MCP Client 経由で動作を確認

## 備考

- トランスポートの種類（stdio / SSE / HTTP Streaming）は接続方式の違い。初期設定では stdio が多い。詳細は MCP Server 側のドキュメントに委ねる
- user スコープ（`~/.claude.json` など）は自分のマシン全体に有効、project スコープ（`.cursor/mcp.json` など）はリポジトリ内でチームと共有できる
- VS Code（GitHub Copilot Chat 経由）の `mcp.json` は 2025 年以降に追加された機能であり、バージョンによって対応状況が異なる場合があります
