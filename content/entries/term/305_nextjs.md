# Next.js

## ひとことで
Next.js は、React を使って本格的なWebアプリを作るためのフレームワーク。画面作りだけでなく、ルーティング、サーバー処理、最適化なども扱う。

## 何をしてくれるか
React はUIを作る中心的な道具だが、アプリとして公開するには他にも必要なものが多い。Next.js は、その周辺機能をまとめて扱いやすくする。

たとえば、ページ構成、データ取得、画像最適化、サーバー側の処理、ビルド設定などが関係する。

## 実務での意味
Next.js が出てくると、単なる画面部品の話から、Webアプリ全体の構造の話に広がる。

AIに修正を頼む場合も、Reactコンポーネントだけでなく、ファイル配置、ルーティング、サーバー側処理、デプロイ先まで関係することがある。

## 非エンジニアのつまずき
React と Next.js の違いがわかりにくい。

ざっくり言うと、React は画面部品を作る道具。Next.js は React を使ってWebアプリ全体を組み立てるための枠組み。

また、Next.js はバージョンや設計方針によって書き方が変わることがある。古い記事のコードをそのまま使うと、今のプロジェクトに合わない場合がある。

## 最初にどこまで理解すればよいか
- Next.js は React の周辺を含めたフレームワーク
- ページ、ルーティング、サーバー処理、最適化が関係する
- Vercel と一緒に語られることが多い
- バージョン差分やプロジェクト構成に注意する

## 私のコメント
Next.js は、AIでアプリを作るときにかなり頻繁に出てくる。便利な一方で、ファイル配置やルールを知らないと、AIがそれっぽいが動かないコードを作ることがある。図鑑では「Reactの上にあるアプリ制作の枠組み」として押さえたい。

## 誌面デザイン案
ワークフロー型。`React components` から `routing`、`server logic`、`build`、`deploy` へ流れる図。
