# JavaScript

## ひとことで
JavaScript は、Webページに動きや処理を加えるために広く使われるプログラミング言語。現在はブラウザだけでなく、サーバーや開発ツールでも使われる。

## 何をしてくれるか
Webサイト上のボタン操作、画面の切り替え、入力チェック、データ取得など、利用者の操作に反応する部分を作れる。

React や Next.js も JavaScript の世界の上にある。TypeScript も JavaScript を土台にしているため、周辺用語の中心にある言語として見ると整理しやすい。

## 実務での意味
AIにフロントエンドの修正を頼むとき、JavaScript の理解があると「画面の見た目」「操作したときの動き」「データの扱い」を分けて話しやすくなる。

非エンジニアにとっては、コードを全部書ける必要はない。まずは、JavaScript が画面やアプリの動きを担当することが多い、と押さえればよい。

## 非エンジニア視点のつまずき
JavaScript と Java は名前が似ているが、別の言語として扱う。ここを混ぜると会話が崩れやすい。

また、同じ JavaScript でも、ブラウザで動く話、Node.jsで動く話、Reactの中で書く話がある。会話では「どこで動くJavaScriptか」を確認するとよい。

## 最初にどこまで理解すればよいか
- Webの動きを作る中心的な言語
- TypeScript、React、Next.jsの土台
- ブラウザでもサーバー側でも使われることがある
- Javaとは別物

## 私のコメント
JavaScript は周辺語彙が多いので、最初から全部理解しようとすると疲れる。図鑑では「中心にある言語」として置き、React や TypeScript などは枝として辿れるようにしたい。
