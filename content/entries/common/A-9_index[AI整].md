# 索引

## tagline

気になる用語を名前から逆引きするための案内ページです。アルファベット順・五十音順の 2 系統を収録しています。

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## 何をしてくれるか

本書の巻末に収録する逆引き索引です。英字略称（GPT・MCP・RAG など）はアルファベット順に、日本語名・カタカナ名（「コンテキスト」「プロンプト」など）は五十音順に並べ、各見出しには ID とページ番号を添えています。

## どこで出会うか

「ChatGPT と GPT-4 は別物なの？」「さっき出てきた RAG はどこに書いてあった？」と感じたときに開くページです。電子版では見出しをクリックするとそのエントリに直接ジャンプします。

## メイン図

### 図の狙い

索引の 2 系統（アルファベット順・五十音順）と、1 件の見出しが ID・ページ番号とセットになっている構造を 1 枚で示します。

### C. 概念図（figure_type: structure）

- 中心に置く概念: 索引（逆引きの入口）
- 周辺の要素: アルファベット順索引 / 五十音順索引 / letter 順目次 / 各見出し（ID＋ページ番号）/ 愛称・表記ゆれからの橋渡し
- 関係の描き方: 索引を中心に各エントリへ矢印が伸びる放射状

## この用語の見どころ

### 1. 役割

アルファベット順と五十音順の 2 系統で用語を逆引きできます。

### 2. うれしさ

略称・愛称・正式名のどこから引いても目的エントリに辿り着けます。

### 3. 注意点

愛称（Nano Banana 等）は正式名エントリに統合しているため索引に橋渡し注記があります。

### 4. どこで役立つか

業界用語の表記ゆれが多い AI 開発領域での語彙確認に役立ちます。

### 5. はじめに

各見出しに ID とページ番号が付く、という仕組みを把握しておきます。

### 6. 深掘り先

この本の読み方（A-2）、略称表記（A-11）

## 開発フローでの位置（必須）

1. 用語に出会う — 会話・記事・AI 応答の中で気になる語が出てくる
2. 索引で引く — アルファベット順か五十音順で見出しを探す
3. ID を確認する — 見出し横の ID（例：B-3）とページ番号で場所を特定する
4. エントリに飛ぶ — 電子版はクリック、紙版はページ番号で開く
5. 関連語を辿る — エントリ下部の関連用語ピルで次の語へ広げる
