# Figma

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## tagline

ブラウザで動くクラウド型の UI デザインツールです。チームで同時に編集でき、デザインから実装まで一貫して使えます。

## 何をしてくれるか

画面レイアウトのデザインと共同編集を担います。複数人のカーソルが見える状態でファイルを編集でき、Dev Mode（デブモード）から CSS など実装向けコードも取り出せます。

## どこで出会うか

デザイナーや PM（プロダクトマネージャー）が UI の仕様書代わりに使う場面で名前を見ます。「Figma の URL 送ります」と渡されることが多く、開発者はそれを見て実装します。AI に UI 生成を頼む起点にもなります。

## メイン図

### 図の狙い

Figma ファイルを AI に渡すと、デザインが即コードになる流れを示します。

### B. 登場シーン（figure_type: structure）

- シーン1: デザイナーが画面レイアウトを Figma で作成し、URL をチームに共有する
- シーン2: 開発者が Dev Mode を開き、CSS やコンポーネント名を確認して実装する
- シーン3: AI に「この Figma の Dev Mode の CSS を渡す」と伝え、コンポーネントを生成させる
- 並べる基準: 役割（デザイナー → 開発者 → AI）の順で、ファイルが橋渡しになる視点


## 会話での使い方例

「Figma の Dev Mode の CSS を Claude に渡したら、コンポーネントが一発で組めました。」

## この用語の見どころ

### 1. 役割

UI デザインの作成・共有・実装連携を担うクラウドツールです。

### 2. うれしさ

Dev Mode で CSS が取れるため、AI へのコード生成指示が具体的になります。

### 3. 注意点

共有リンクの公開範囲設定が誤ると、社外から見えない・見えすぎるトラブルが起きます。

### 4. どこで役立つか

デザインと実装の引き継ぎ・AI による UI 生成の起点として役立ちます。

### 5. はじめに

View 権限と Edit 権限の違い、Dev Mode の存在を押さえます。

### 6. 深掘り先

Dev Mode、Auto Layout、Figma Make


## 開発フローでの位置（必須）

1. 要件定義 — どんな画面が必要かをチームで整理する
2. デザイン作成 — Figma でレイアウト・コンポーネントを組む
3. 共有・確認 — URL を渡して関係者がコメント・承認する
4. 実装連携 — Dev Mode の CSS やコンポーネント名を AI に渡してコード生成する
5. 修正ループ — 実装結果をデザインと照合し、Figma 上で仕様を更新する

## コミュニティ補完メモ

- Excalidraw（B-31）との住み分け: Excalidraw はホワイトボード的な手書きラフ用途。Figma は仕様書・実装連携まで担う本番デザインツール
- Canva（B-33）との住み分け: Canva はバナー・SNS 投稿など非エンジニア向けグラフィック用途。Figma はプロダクト UI 設計が主眼
- Dev Mode は有料プランで一部機能が制限される（2026-04-30 時点）。時変情報のため出版前に要確認
