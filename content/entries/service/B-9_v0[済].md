# v0

## tagline

Vercel が提供する UI 生成 AI です。テキスト指示で React＋Tailwind のコンポーネントを出力します。

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## 何をしてくれるか

「カード一覧を作って」と入力すると、そのままブラウザで動く React コンポーネントのコードを生成します。プレビューをその場で確認し、コードをコピーして自分のプロジェクトへ貼り付けて使えます。

## どこで出会うか

Vercel（ヴァーセル）のサービス紹介や、フロントエンド開発の入門記事でよく出てきます。「デザインを言葉で伝えたらコードになった」という体験談として SNS や技術ブログにも登場します。

## メイン図

### 図の狙い

プロンプト入力からプレビュー確認・コード取得・デプロイまでの流れを 1 画面で完結させる様子を示します。

### C. 概念図（figure_type: workflow）

- 中心に置く概念: v0 のチャット画面
- 周辺の要素: プロンプト入力 → UI プレビュー → コードコピー → Vercel デプロイ
- 関係の描き方: 左から右への矢印フロー、各ステップに人物の吹き出しを添える

## 会話での使い方例

「v0 で雛形を作って Next.js に貼り付けると早いですよ。」

## この用語の見どころ

### 1. 役割

テキスト指示から React UI コードを生成するサービスです。

### 2. うれしさ

デザインの知識なしでも動くコンポーネントをすぐ試せます。

### 3. 注意点

生成コードは React 前提のため、他フレームワークへの転用に手間がかかります。

### 4. どこで役立つか

プロトタイプや LP のパーツを素早く作りたい場面で効果が出ます。

### 5. はじめに

v0.dev にアクセスし、日本語で画面の要望を書くだけで始められます。

### 6. 深掘り先

Vercel（B-20）、Next.js（F-11）、shadcn/ui

## 開発フローでの位置（必須）

1. プロンプト入力 — チャットに作りたい画面の要望を書きます
2. プレビュー確認 — ブラウザ上で UI を確かめて修正を伝えます
3. コード取得 — 満足したらコードをコピーして手元に貼ります
4. Next.js へ組み込み — shadcn/ui 準拠で馴染みやすい構造です
5. Vercel へデプロイ — そのまま公開して動作確認を行います

## コミュニティ補完メモ

- Vercel（B-20）との住み分け：v0 は Vercel が提供する UI 生成 AI サービス。Vercel 本体（ホスティング・デプロイ基盤）の詳細は B-20 で扱います
- Next.js（F-11）との住み分け：v0 が生成するコードは Next.js プロジェクトへの組み込みを前提とすることが多いが、Next.js 自体のフレームワーク詳細は F-11 で扱います
- Bolt.new（B-11）との住み分け：どちらも「AI で画面を作る」サービスだが、v0 は Vercel 生態系＋ React コンポーネント特化。Bolt.new はフルスタックアプリ生成寄りで住み分けが分かれます
- Cursor（B-4）との住み分け：Cursor はエディタ内で AI を使うアプローチ。v0 はブラウザ上で完結する UI 生成専用サービスで、生成したコードを Cursor に持ち込む組み合わせもあります
