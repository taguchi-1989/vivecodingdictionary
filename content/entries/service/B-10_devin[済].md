# Devin

## tagline

Cognition 社の自律型 AI エンジニアです。タスクを渡すと人の介在なく動き続けます。

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## 何をしてくれるか

「このバグを直して Push して」と指示すると、調査・実装・テスト・PR（プルリクエスト）作成までを自律で進めます。人が傍で操作する補助型ではなく、自分で動き続けるエージェント型です。

## どこで出会うか

「AI に任せたら勝手にコードを書いてくれた」という記事や SNS の感想で名前が出てきます。AI エージェント（自律型 AI）の先行事例としてよく引用され、SWE-Bench のスコアで注目を集めたサービスです。

## メイン図

### 図の狙い

タスク指示から完了報告までを Devin が独立して走り抜ける流れを示し、「補助型」との違いを直感で掴んでもらいます。

### C. 概念図（figure_type: workflow）

- 中心に置く概念: 自律ループ（調査→実装→テスト→PR）
- 周辺の要素: 人間（依頼側）/ Devin / リポジトリ / テスト結果 / レビュー待ち PR
- 関係の描き方: 人が左端でタスク指示 → Devin が中央でループ → 右端でレビュー待ち PR を人に返す

## 会話での使い方例

「Devin に Issue ごと渡したら、PR まで自律で上げてきたのが新鮮でした。」

## この用語の見どころ

### 1. 役割

タスク指示を受けて自律的にコードを書き PR を上げるエージェントです。

### 2. うれしさ

依頼後は放置でき、人の作業を別タスクに集中させられます。

### 3. 注意点

料金が高くエンタープライズ向け中心のため個人利用は限定的です。

### 4. どこで役立つか

仕様が明確なバグ修正や小機能追加で効果が出やすいです。

### 5. はじめに

補助型（Cursor・Claude Code）との違いと自律型の意味を押さえます。

### 6. 深掘り先

SWE-Bench、自律エージェント、Claude Code

## 開発フローでの位置（必須）

1. タスク定義 — Issue や Slack でやることを明確に伝えます
2. 自律実行 — Devin がリポジトリを読み、実装・テストを進めます
3. 進捗確認 — Devin がチャットで途中経過をレポートしてきます
4. レビュー — 人が PR の差分を確認して承認・差し戻しを判断します
5. マージ — 問題なければ人の手でマージして完了です

## 誌面ポンチ絵メモ

### メイン図（左ページ中段 / figure_type: workflow）

- 描く内容: 左端に人物（依頼者）がタスクカードを持つ。中央に Devin アイコンが「調査→実装→テスト→PR」の 4 ステップを矢印でループする。右端に「PR 完成」ラベルが人物に届く構図
- 登場人物: 依頼者（ノートPC を持つオフィスワーカー）と Devin（ロボットアイコン）
- 吹き出し・心の声: 依頼者「Issue 渡すだけでいいの？」/ Devin「調査中…実装中…テスト完了。PR 上げました。」
- 中央キーワード: 「自律ループ」

### 6 視点アイコン（右ページ上段）

- 共通アイコン流用（個別演出が要るときだけ書き足す）

### 開発フロー図（右ページ下段）

- Step 1 のアイコン/絵柄: タスクカード — タスク定義
- Step 2 のアイコン/絵柄: ギア回転 — 自律実行
- Step 3 のアイコン/絵柄: チャット吹き出し — 進捗レポート
- Step 4 のアイコン/絵柄: 差分ビュー — レビュー
- Step 5 のアイコン/絵柄: チェックマーク — マージ
- 矢印で示す流れの意図: 人が関与するのは両端（依頼とレビュー）だけで、中央は Devin が完結させることを示す


## コミュニティ補完メモ

- Cursor（B-4）・Claude Code（B-7）との住み分け：Cursor と Claude Code は「人が操作する隣で AI が補助する補助型」。Devin は「人が離れていても自律でタスクを完結させる自律型」。この補助型 vs 自律型の違いが本エントリの核心です
- SWE-Bench（E-1）との関係：Devin が初期注目を集めた根拠であるベンチマークです。スコアの詳細は E-1 側で扱い、本エントリはサービスの使い方に絞ります
- Bolt.new（B-11）との住み分け：Bolt は UI／フロントエンドの自動生成が中心。Devin はリポジトリを横断したバックエンド実装・バグ修正まで扱う点が異なります
