# 決定論的／非決定論的

<!-- ━━━━━━━━ 左ページ ━━━━━━━━ -->

## tagline

同じ入力で毎回同じ出力が返るか、揺らぐかという性質の区別です。

## 何をしてくれるか

決定論的（deterministic）は同じ入力に常に同じ出力を返す性質、非決定論的（non-deterministic）はその逆です。LLM（大規模言語モデル）は基本的に非決定論的で、temperature（多様性パラメータ）を 0 にしても完全な再現は保証されません。

## どこで出会うか

「同じ質問なのに昨日と違う答えが返った」という体験がこれにあたります。テスト自動化やコード生成では temperature=0 と seed を組み合わせて出力を安定させます。

## メイン図

### 図の狙い

同じ入力ブロックから、決定論的なケース（1 本の矢印で同じ出力）と非決定論的なケース（複数の矢印でそれぞれ異なる出力）を並べて、違いを直感的に示す。

### B. 登場シーン（figure_type: comparison）

- シーン1: 利用者が「同じ質問」を 3 回送る
- シーン2: 決定論的側では毎回「A」が返る
- シーン3: 非決定論的側では「A」「B」「C」と揺らぐ
- 並べる基準: 左右に決定論的 vs 非決定論的を配置

## 会話での使い方例

「LLM は非決定論的なので、seed 固定しないとテスト結果が毎回変わります。」

## この用語の見どころ

### 1. 役割

出力の再現性があるかどうかを表す性質の区別です。

### 2. うれしさ

非決定論と知っていれば「答えが変わるのはバグではない」と判断できます。

### 3. 注意点

temperature=0 でも完全な再現性は保証されないことがあります。

### 4. どこで役立つか

テスト自動化やコード生成で出力を安定させたいときに役立ちます。

### 5. はじめに

「LLM は非決定論的」という前提と temperature の役割を押さえると十分です。

### 6. 深掘り先

temperature、seed パラメータ、System Prompt


## 開発フローでの位置（必須）

1. 仕様・要件確認 — 出力の再現性が必要かどうかを確認します
2. パラメータ設定 — temperature=0 と seed 値を API に渡します
3. 動作確認 — 同じ入力で複数回呼び出し、出力の揺れを確認します
4. 本番判断 — 揺れが許容できないときは seed 固定を徹底します

## 備考

- temperature=0 でも GPU の並列計算や浮動小数点演算の順序依存で微妙な差異が出ることがある（モデルや推論基盤による）
- seed パラメータは OpenAI API では対応しているが、全サービス・全モデルで使えるわけではない（2026-04-30 時点）
- 「決定論的」という語は数学・計算理論・ソフトウェア工学で共通の概念であり、LLM 固有の用語ではない
